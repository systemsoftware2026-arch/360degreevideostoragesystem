# 360-Video Storage Optimization via DRL & Greedy Algorithm
**제한된 스토리지 용량 내 비디오 품질 향상을 위한 강화학습 기반 360도 비디오 타일 버전 결정 알고리즘 및 시뮬레이션**

본 저장소는 서버의 저장 공간 제약 조건 하에서 사용자의 시청 품질(QoE)을 극대화하기 위해, 360도 비디오의 최적 타일 조합을 결정하는 심층 강화학습(DRL) 및 그리디(Greedy) 알고리즘 기반의 시뮬레이션 프로그램을 포함하고 있습니다.

## 📌 주요 특징 (Key Features)
* **타일 기반 최적화**: 360도 비디오를 여러 타일(Tile)로 분할하여 관리하며, 각 타일의 다중 품질 버전 중 최적의 조합을 선택합니다.
* **DRL (Deep Reinforcement Learning) 기반 의사결정**:
    * **Dueling DQN** 알고리즘을 사용하여 복잡한 타일 선택 문제를 해결합니다.
    * 사용자의 네트워크 대역폭 분포, FoV(Field of View), 타일별 파일 크기 및 품질 정보를 상태 공간(State Space)으로 정의합니다.
* **Greedy 알고리즘 보정**: DRL 모델이 제안한 후보군과 비디오 인기도를 결합하여, 전체 스토리지 용량 한도를 준수하는 최종 타일 조합을 산출합니다.
* **현실적인 시나리오 모델링**: 마르코프 의사 결정 프로세스(MDP)를 통해 에이전트와 환경의 상호작용을 모델링하고 보상을 계산합니다.

## 🚀 사용 방법 (Usage)

1.  **환경 준비**:
    * 본 프로젝트의 압축을 해제한 후, 루트 디렉토리에 `stateinfo` 폴더를 생성합니다.
2.  **모델 학습 (Training)**:
    * `dqn5.py` 파일의 메인 함수에서 `isTrain` 변수를 `1`로 설정합니다.
    ```
    python dqn5.py
    ```
3.  **의사결정 및 추론 (Inference)**:
    * `dqn5.py` 파일의 메인 함수에서 `isTrain` 변수를 `0`으로 변경합니다.
    ```
    python dqn5.py
    ```
4.  **최종 결과 도출**:
    * `seg_storage_greedy.py` 명령어를 사용하여 최종 결과를 얻습니다.
    ```
    python seg_storage_greedy.py
    ```
